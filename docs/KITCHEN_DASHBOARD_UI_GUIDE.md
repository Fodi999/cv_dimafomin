## 🎨 Цветовая схема

| Статус | Цвет | Иконка | Анимация |
|--------|------|--------|----------|
| planned | 🟡 Yellow | pending | Статична |
| cooking | 🔵 Blue | pulse | Пульсирует 💫 |
| completed | ✅ Green | checkmark | Пульсирует 💫 |

## 🔄 State Transitions (User Flow)

```
START
  ↓
[Open /recipes]
  ↓
load menu from backend
  ↓
filter by status
  ↓
show 3 sections (planned, cooking, completed)
  ↓
USER CLICKS "Начать готовить"
  ↓
POST /api/menu/:id/start
  ↓
toast: "✅ Начали готовить!"
  ↓
reload menu
  ↓
card moves: planned → cooking
  ↓
USER CLICKS "Готово!"
  ↓
POST /api/menu/:id/complete
  ↓
toast: "🎉 Отлично!"
  ↓
reload menu
  ↓
card moves: cooking → completed
  ↓
END
```

## 🎬 Demo Sequence

**Начальное состояние:**
```
📊 Stats: planned: 0, cooking: 0, completed: 0
🚫 Все секции скрыты (нет рецептов)
💬 Сообщение: "Ваше меню пусто"
```

**После добавления рецепта:**
```
📊 Stats: planned: 1, cooking: 0, completed: 0
✅ Секция "Сегодня готовим" видна
🎨 1 карточка с кнопкой "Начать готовить"
```

**После нажатия start:**
```
📊 Stats: planned: 0, cooking: 1, completed: 0
🔵 Секция "Готовится" раскрыта
⏱️  Показывает время начала
🎨 Карточка с кнопкой "Готово!"
```

**После нажатия complete:**
```
📊 Stats: planned: 0, cooking: 0, completed: 1
✅ Секция "Приготовлено сегодня" видна
📝 Показывает время завершения
🎨 Карточка readonly
```
